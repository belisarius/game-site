AutotHotKey and Localization

Tuesday, February 18, 2014

Many years ago, one of my co-workers at Aksys introdced me to a piece of software called AutoHotKey. The simple description of AHK is that it's a very simple scripting language that gives you simple access to a lot of stuff, most importantly stuff like keypresses.

After discovering AHK, I've found it to be indispensible for my localization work. There are a lot of tasks that it can easily automate, which speeds up my work and reduces the chance for errors. For instance, here's one of the scripts I wrote for my current project:

#j::
SendInput, ^{Left}
Return

Basically, when I press Windows+j, AHK tells windows I pressed Control+←. This jumps the cursor to the beginning of the current word. That might not seem all that interesting in and of itself (after all, I could just press Control+←), but I combine it with two other scripts that make it much more useful in the context of localization editing.

As anyone who's done video game localization knows, practically every project will have character limits. As you might guess, these are limits on how many characters (i.e. letters, numbers, punctuation marks, etc) you can have in a given section of text. Sometimes, especially when a game uses a variable-width font this is based soley on space, and you often have to eyeball how much space you have (always stressful), but much more frequently you're given a set number that defines how many characters you can have per line.

Since most translation and editing is done in Excel, sometimes you can just use LEN() and get the number of characters that way, but usually multiple lines are squished into a cell, and then LEN() won't work, since it just counts all the characters in the cell. You can count characters by pressing → and counting how many times, but since character limits can range between 20 and 50 characters, that gets old fast. Plus, once you've counted all your characters, you need to insert something that indicates a new line, whether that's a hard return or something like /n.

Enter AutoHotKey!

; This is for counting characters (38) in this case.
#h:: ; This sets the shortcut key combination—in this case, it's Windows key (#) and h.
SendInput, {Right 38} ; "SendInput" basically means "send this key or key combination". "{Right 38}" means the right arrow key, and to send it 38 times (the number of characters I'm allowed per line.)
Return ; End each snippet with "Return" or it'll just keep running.

; If the character-counting snippet ends in the middle of a word, I use this to jump to the beginning of that word.
#j::
SendInput, ^{Left}
Return

; This inserts a hard return. I customised this script for Excel, and in Excel you insert a carriage return with alt + Enter. The "!" means the alt key, and {Enter} means the Enter key. 
; If you wanted to insert, say, "/n" as a control code for carriage returns, you'd customize the second line to look like this: "Send, /n"
#n::
Send, !{Enter}
Return

With this collection of commands, I can easily count characters and then insert carriage returns (or control characters) when needed, using a collection of three keys—h, j, and n. This makes the process of checking a line after its been written far simpler.

The syntax for a script like this is fairly simple:

#h::

As the name would suggest, pretty much everything you do with AHK is triggered using hotkeys, so the first part of every script is defining one. In this case, there are three parts:

  • # This is a modifier, in this case the Windows key. Generally modifiers are for the "special" keys, i.e. alt, shift, control, and Windows. Putting one next to another key means you're sending a signal that's the modifier plus that key. For instance, ^v would be the command you know as "paste." Modifiers only affect the very next key.
  • h This is the key I'm using for this definition.
  • :: These just mean "We've finished defining our hotkey, shit is about to get real."
SendInput, {Right 38}

This is the meat of the snippet, and it's essnetially comprised of two parts:

  • SendInput This is just a command that tells AutoHotKey "Whatever comes after this is what I want you to send to Windows." There are actually five different send types (Send, SendRaw, SendInput, SendPlay, SendEvent). Generally we'll use "SentInput" becuase it's fast and buffers mouse and keyboard input when its triggered (which means it can't be interrupted).
  • {Right 38} Here is what we're sending. In this case, it's →, 38 times. Enclosing "Right" in brackets tells AutoHotKey that it should treat it as a command of some sort, and not just print "{Right}". If you do just want to print text, that's even easier: The line SendInput, I'm faster than anybody! would spit out "I'm faster than anybody."
Return

Return should always appear the end of your snippet—it's what tells AHK "All right, we're done here, stop doing stuff." Without it, AHK will continue to execute. For instance, if I forgot to put Return at the end of my character counting snippet, once it had finished moving the cursor 48 times, it would then jump the cursor to the beginning of the current word (as per the next snippet).

You can use this basic fomula to automate just about any repetetive task. I've also used it to quicky insert character names so that I don't run the risk of messing up the spelling.
